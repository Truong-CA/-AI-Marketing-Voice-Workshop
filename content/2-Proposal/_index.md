# AI Marketing Voice — AI-Powered Marketing Script & Voice Generation Platform on AWS

### 1. Project Summary
**AI Marketing Voice** is a web application that helps small businesses and content creators quickly generate marketing scripts for short videos and ready-to-use Vietnamese voiceovers for social media platforms (TikTok, Instagram, Facebook, YouTube). Users simply describe their product; the system uses **Amazon Bedrock** with the Claude 3 Sonnet model to generate structured scripts with hooks, body content, CTAs, and hashtags, combined with **VieNeu-TTS** — an open-source Vietnamese text-to-speech model running directly on the backend server to synthesize voiceovers, which are then stored in **Amazon S3** and distributed via **Amazon CloudFront**. The platform includes an account system (email/password + Google login), automated email notifications via **Amazon SES**, operational logging via **Amazon CloudWatch Logs**, and a credit-based billing mechanism to fairly control costs for each user.

### 2. Problem Statement
**What is the problem?**
Small businesses and individual content creators often lack the time, copywriting skills, or budget to write platform-specific scripts and record professional Vietnamese voiceovers for every product they want to promote. Hiring a copywriter or voice actor for each short video is both slow and costly; popular cloud TTS services do not yet provide adequate support for natural Vietnamese voices.

**Solution**
AI Marketing Voice allows users to enter basic product information (name, description, price, target audience, platform, tone, duration, call-to-action) and immediately receive:
- A script generated by a large language model.
- A **Vietnamese** audio file of that script, selected from 10 diverse voice options (male/female, Northern/Southern accent, natural/storytelling/news style).

All within a single request-response flow, at a fraction of the cost and time compared to manual production.

### 3. Solution Architecture

**AWS Services Used:**
- **Amazon Bedrock** — calls the Claude 3 Sonnet foundation model to generate structured marketing scripts in JSON format.
- **Amazon S3** — stores generated audio files (from VieNeu-TTS) and serves as the origin for CloudFront.
- **Amazon CloudFront** — CDN in front of S3, distributing audio files faster and reducing direct access load on the bucket.
- **Amazon SES** — sends automated notification emails when a new user registers an account.
- **Amazon CloudWatch Logs** — collects backend application logs (registration, login, script generation, audio creation, system errors) for monitoring and debugging in production.
- **AWS IAM** — least-privilege permissions for the backend to call Bedrock, S3, SES, and CloudWatch Logs.

**Voiceover:** The system uses **VieNeu-TTS**, an open-source Vietnamese text-to-speech model licensed under Apache 2.0, chosen because commercial TTS services lack adequate Vietnamese voice support.

![System Architecture](https://truong-ca.github.io/-AI-Marketing-Voice-Workshop/images/main.png)

**Component Design:**
- **Frontend** — collects product information from the user, calls the backend API, selects a voice, plays and downloads the generated audio.
- **Backend** — validates input data, builds the LLM prompt, calls Bedrock, calls VieNeu-TTS to synthesize speech, uploads audio to S3 served via CloudFront, sends welcome emails via SES, writes logs via CloudWatch Logs, and manages user accounts and the credit ledger.
- **Prompt Layer** — a dedicated prompt builder maps the platform (TikTok/Instagram/Facebook/YouTube), tone, and duration into a structured prompt, along with a validator that checks required fields before calling Bedrock.
- **Credit System** — each new registration receives welcome credits; each script generation and voice creation deducts the corresponding credits, with all transactions recorded for easy auditing.

### 4. Technical Implementation
**Implementation Phases**
- REST API design and data modeling — Weeks 1–2.
- Build prompt builder + validator for script generation — Week 3.
- Integrate Amazon Bedrock for script generation and test with sample products — Week 4.
- Integrate VieNeu-TTS + Amazon S3 + CloudFront for audio generation and distribution — Week 5.
- Add authentication, credit system, and notification emails via SES — Weeks 6–7.
- Build and connect the React frontend with the API — Weeks 8–9.
- Add CloudWatch Logs, testing, tighten IAM, and cost cleanup procedures — Weeks 10–12.

**Technical Requirements**
- AWS account with access granted to the Claude 3 Sonnet model in Amazon Bedrock.
- An S3 bucket for storing generated audio, and a CloudFront distribution pointing to that bucket.
- A verified email address in Amazon SES for sending welcome emails.
- A CloudWatch Log Group to receive logs from the backend.
- An IAM role/user with minimum permissions for `bedrock:InvokeModel`, `s3:PutObject`/`s3:GetObject` scoped to the target bucket, `ses:SendEmail`, and `logs:CreateLogGroup`/`logs:CreateLogStream`/`logs:PutLogEvents`.
- Python 3.10+, FastAPI, boto3, SQLAlchemy, the `vieneu` library (local TTS); Node.js + React for the frontend.

### 6. Budget Estimate

Infrastructure costs: 300 script generation requests/month, 300 audio generation requests/month

- Amazon Bedrock (Claude 3 Sonnet): $2.97/month (300 model calls, averaging 800 input + 500 output tokens per call).
- Amazon S3 Standard: $0.01/month (0.3 GB audio storage, 300 PUT requests).
- Amazon CloudFront: $0.05/month (0.6 GB data transfer out, 600 play/download requests).
- Amazon SES: $0.00/month (within the free tier of 3,000 emails/month for the first 12 months).
- Amazon CloudWatch Logs: $0.00/month (within the free tier of 5 GB logs/month).
- AWS IAM: $0.00/month (free).
- **Total: $3.03/month, $36.36/12 months**

### 7. Risk Assessment
- **Model returns incorrect JSON format** — mitigated by enforcing strict format constraints in the prompt and validating on the backend, returning clear error messages.
- **Cost overruns due to unlimited usage (Bedrock)** — mitigated by the credit system, limiting usage per user.
- **AWS credentials exposure** — mitigated by using environment variables and applying IAM least-privilege.
- **VieNeu-TTS running on CPU is slower than cloud TTS services** — the speed trade-off is accepted in exchange for zero cost and no third-party dependency; caching is applied for voice preview samples to reduce the number of model re-runs.
- **SES in Sandbox mode can only send to verified emails** — Production access must be requested before a wide release to real users.

### 8. Expected Outcomes
Successfully build a reasonably complete system that allows users to enter product information, automatically generate marketing scripts using Amazon Bedrock, convert scripts into Vietnamese voiceovers with 10 voice options using VieNeu-TTS, and download audio files via CloudFront. The system operates stably, is secured according to the least-privilege principle, logs comprehensively using CloudWatch, and can have all AWS resources easily cleaned up upon completion.
