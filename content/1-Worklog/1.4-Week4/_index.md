### Week 4 Objectives

* Explore storage services on AWS.
* Enhance Amazon EC2 management skills.
* Practice using Amazon S3 for data storage and management.

### Tasks to Be Completed This Week

| Day | Tasks | Start Date | Completion Date | Reference |
| --- | ----- | ---------- | --------------- | --------- |
| Mon | Review Amazon EC2, explore Instance Types, EBS Volumes, and Snapshots. | 11/05/2026 | 11/05/2026 | https://docs.aws.amazon.com/ec2/ |
| Tue | Practice creating, starting, stopping, restarting, and terminating EC2 Instances; manage Key Pairs and Security Groups. | 12/05/2026 | 12/05/2026 | https://cloudjourney.awsstudygroup.com/ |
| Wed | Study Amazon S3, Buckets, Objects, Storage Classes, and Versioning. | 13/05/2026 | 13/05/2026 | https://docs.aws.amazon.com/s3/ |
| Thu | Practice creating Buckets, uploading/downloading Objects, configuring Bucket Policies, and managing access permissions. | 14/05/2026 | 14/05/2026 | https://docs.aws.amazon.com/s3/ |
| Fri | Study S3 Versioning, Lifecycle Rules, and Best Practices when using Amazon S3. | 15/05/2026 | 15/05/2026 | https://docs.aws.amazon.com/s3/ |

### Week 4 Results Achieved

* Understood the role of Amazon EC2 in deploying and operating servers on the AWS platform.

* Proficiently performed EC2 management operations:
  * Launched, started, stopped, and terminated EC2 Instances.
  * Managed Security Groups and Key Pairs.
  * Created Snapshots for EBS Volumes.

* Understood the storage architecture of Amazon S3 and its components:
  * Bucket
  * Object
  * Storage Class
  * Versioning

* Successfully performed operations on Amazon S3:
  * Created Buckets.
  * Uploaded and downloaded Objects.
  * Managed folders and files.
  * Configured Bucket Policies to control access permissions.

* Understood the storage tiers of Amazon S3 and learned how to select the appropriate Storage Class for each use case.

* Studied the Versioning and Lifecycle Rule mechanisms to effectively manage data and optimize storage costs.

* Gained knowledge of basic security principles when using Amazon S3, including access permission management, restricting public Bucket access, and protecting data on the Cloud.

* Developed the ability to combine Amazon EC2 and Amazon S3 to build a storage and data processing environment for applications on AWS.
