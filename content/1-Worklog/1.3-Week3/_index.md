### Week 3 Objectives

* Learn about network architecture on the AWS platform.
* Understand how to build a networking environment using Amazon Virtual Private Cloud (Amazon VPC).
* Become familiar with networking components and traffic control mechanisms in AWS.

### Week 3 Achievements

* Gained an understanding of AWS networking architecture and the role of Amazon VPC in building isolated virtual networks.

* Learned the purpose and functionality of the following core networking components:
  * VPC
  * CIDR Block
  * Public Subnet
  * Private Subnet
  * Availability Zone

* Understood the functions of Route Tables, Internet Gateways, and NAT Gateways in routing network traffic.

* Identified the differences between Security Groups and Network ACLs, including their scope of operation and security capabilities.

* Successfully created a basic VPC, configured Subnets, and set up Security Groups to allow an EC2 instance to access the Internet.

* Developed foundational skills in designing network architectures for applications deployed on AWS.
